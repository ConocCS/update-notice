# リリース告知・運用ガイドライン
このドキュメントは、CONOCのシステムアップデートにおける「告知文の作成」「HTMLへの反映」「プルリクエストによる確認依頼」の手順を定めたものです。

### 1. 告知文の作成ルール
ユーザーに「何が改善されたか」を正確に伝えるため、以下の構造で作成します。

カテゴリの定義
【機能追加】: 新しい機能の実装、または既存機能の大幅な拡張。
【パフォーマンス改善】: 読み込み速度の向上や、動作の最適化。
【不具合修正】: エラーの解消や、表示の不整合の修正。

記載のポイント
専門用語を避ける: 「N+1の解消」→「表示速度の高速化」など、ユーザーメリットに変換する。
操作感を具体的に: 「計算処理の改善」→「Enterキー押下で粗利計算結果を反映」など。

### 2. 安全な更新ワークフロー
本番環境への誤反映を防ぎ、過去の履歴を管理するために以下の手順を推奨します。

基本原則：直接編集の禁止
本番用ファイルを直接書き換えるのではなく、**「テンプレートをコピーして新規作成」**する運用を徹底してください。

#### 更新の手順
ブランチ作成: feature/update-20260408 のように、リリース日を含めた作業ブランチを作成する。
コピー作成: テンプレートとなるHTMLファイルをコピーし、リリース日を名前に含めた新しいファイルを作成する（例: notices/20260408.html）。
内容編集: 新しいファイルに対して、今回のアップデート内容を反映する。
ローカル確認: ブラウザでファイルを開き、デザイン崩れや誤字脱字がないか目視で確認する。
プルリクエスト提出: 修正したファイルをコミットし、関係者にレビューを依頼する。

### 3. HTML更新時のチェックリスト
system_update_notice.html（または各回の告知用ファイル）を更新する際は、以下のデザインルールを遵守してください。

- [ ] ヘッダー: チャコールグレー（#52565e）の背景にロゴがあるか。
- [ ] カラー: アクセントに CONOC Blue（#4A99D0）を使用しているか。
- [ ] 箇条書き:CONOC Blue になっているか（text-2xl）。
- [ ] レスポンシブ: スマホで表示した際に文字が溢れていないか。

# 告知ページ デザインガイドライン (CONOC Design System)
本プロジェクト（system_update_notice.html）で使用されているデザインルールと実装仕様のまとめです。

### 1. カラーパレット (Brand Colors)
ブランドの信頼感と清潔感を表現するため、以下の3色をベースに構成しています。
色
カラーコード
用途

##### CONOC Blue
#4A99D0
アクセント、リストの点、ボーダー、重要タグ、アイコン

##### Charcoal Grey
#52565e
ヘッダー背景、メインテキスト、実行日バッジ

##### White
#ffffff
全体の背景、カードの背景

##### Slate (Sub)
#f8fafc / #f1f5f9
セクションの背景、薄いボーダー

### 2. タイポグラフィ (Typography)
可読性とモダンな印象を両立させる設定です。

フォント: Inter (欧文), Noto Sans JP (和文)
文字間隔: letter-spacing: 0.02em (標準よりわずかに広げることで、洗練された高級感を出しています)
行間: leading-relaxed (1.625) を基本とし、文章にゆとりを持たせています。

### 3. ヘッダー構成 (Header Structure)
スクリーンショットのイメージに基づいた、ブランドアイデンティティを象徴する領域です。

メインエリア: チャコールグレー（#52565e）背景。中央にロゴを配置し、垂直・水平方向ともにセンター合わせ。
アクセントバー: CONOC Blue（#4A99D0）背景。高さ 48px 固定。ヘッダーの下部に配置し、ページ全体の引き締めとブランドカラーの印象付けを行います。

### 4. コンポーネント・ルール (Component Rules)

#### モダンカード (Modern Card)
角丸: rounded-3xl (大きな角丸を採用し、ビジネスツールとしての堅実さと親しみやすさを両立)
影 (Shadow):
通常時：shadow-sm (ごく薄い影でフラットすぎない奥行きを表現)
ホバー時：transform: translateY(-4px) と共に、より深い影を適用するマイクロインタラクションを推奨。
トップライン: カードの最上部に 1.5px の CONOC Blue のラインを入れることで、視覚的な一貫性を保持します。

#### 機能追加リスト (List Items)
箇条書きの点: CONOC Blue (#4A99D0) で通常より大きく表示（text-2xl）。
強調表現: 重要なキーワードや機能名は font-bold を適用し、必要に応じて border-b-2 border-conoc-blue/20（薄い青の下線）で強調します。

### 5. 実装時の注意点

フレームワーク: Tailwind CSS を利用。
ロゴ参照: ロゴ画像は conoc-logo.png を使用。ファイルが見当たらない場合や読み込みエラー時は、SVGによる再現またはテキスト（CONOC）が白文字で表示されるフォールバックを必ず設定してください。
レスポンス: モバイル閲覧を前提とし、パディングやフォントサイズは md: プレフィックスを活用してデバイスごとに最適化します。
